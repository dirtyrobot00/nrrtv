- 시스템 환경이나 값을 수정하는 커맨드는 수행하지 말 것(다만 ls 같이 조회성은 수행가능)
- 그 외에 해당 디렉토리에서의 소스 수정이나 파일 추가는 별도의 질문(허락)없이 수행할 것

- 소스코드를 수정할 때는 항상 git add 를 수행할 것
- 파이썬은 python3을 사용할 것
- 파일이 아닌 DB에 저장할 것을 상정하고 개발할 것
- 모든 프로그램의 수정이나 신규작성은 그 내용을 정리하여 READ_ME_JHLEE.md에 작성할 것
    - 프로그램 사용법(커맨드 등)
- 사용자가 기능을 새로 요청할 때, 이미 작성한 소스코드 프로그램(READ_ME_JHLEE.md참고)을 최대한 활용할 것
    - 모든 파이썬 프로그램은 외부 파이썬에서 실행이 가능하도록 설계할 것
- 코어 프로그램은 src/ 디렉토리, 그걸 한번 래핑하여 수행하는 프로그램은 script/ 디렉토리에 작성할 것

- 해당 프로젝트는 데이터를 주기적으로 수집하는데 그 목적이 있음
- 이미 지난 데이터(backfill) 수집과 실시간 현재데이터(realtime) 두 가지로 구분이 되어있다고 볼 수 있음
- 항상 backfill과 realtime은 쌍으로 이루어져 있어야 하며, 같은 프로그램 다른 명령어 정도로 구현하는게 좋음

- DB에 테이블을 만들때는 DB_TABLE.md에 테이블과 스키마를 정리
    - 추후 테이블이 필요할 때 참고하도록

- 모든 프로그램은 Postgre DB에 저장하도록 개발
